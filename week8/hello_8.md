 - 최적화
        -웹 서비스의 성능을 개선하는 기술
        
        -불필요한 연산을 줄여 렌더링하는 기술
        
        - 리액트: 매우 빠르고, 브라우저의 불필요한 연산 최소화
        - 메모이제이션:연산 최적화를 하는 방법
        - 동적계획법: 불필요한 연산을 줄여 프로그램의 실행 속도 빠르게 하기
        - useMemo:메모지에이션 기법을 사용해 불필요한 함수 호출을 막아주는 것’
            - 특정 함수를 호출했을때 그 함수의 반한값을 기억
            - 다시 호출하면 기억해두었던 값을 반환
            - 최적화 가능
        - React.Memo: 컴포넌트가 불필요하게 리렌더되는 상황 방지 가능
            - 인수로 전달할 컴포넌트를 메모이제이션된 컴포넌트로 만들어 반환
            - 컴포넌트가 크고 복잡할수록 불필요한 렌더링을 방지하면 브라우저의 연산량을 줄여 주어 성능 최적화에 도움을 줌
        - HOC: 교차 컴포넌트(컴포넌트의 기능을 다시 사용하기 위한 리액트의 고급 기술)
        - 횡단 관심사=크로스 커팅 관심사:비즈니스 로직과 구분되는 공ㄷ통 기능을 지칭할 때 사용하는 용어
        - TosoItem: 낱낱의 할 일 아이템을 담당하는 컴포넌트에서 불필요한 렌더링이 일어나는지 확인하고 최적화
            - 개별 아이템 체크박스에서 완료,미완료를 토글할 때가 아니면 리렌더 할 필요 없음
        - 불필요한 함수 재생성 방지
            - useCallback 함수 사용해 할 일 관리 앱 최적화 하기
        - 유의
            - 항상 마지막
            - 모든것을 할 필요는 없음
            - 다시 한번 구조설계 돌아보기
            - 최적화는 여기서 끝나지 않음
